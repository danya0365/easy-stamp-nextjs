## [1.13.0] - 2026-06-20

### Added
- **ยืนยันตัวตน 2 ชั้น (2FA/TOTP) สำหรับผู้ดูแลระบบ** — สแกน QR ด้วยแอป Authenticator (Google Authenticator, Authy, 1Password ฯลฯ) + รหัสสำรอง (recovery codes) · **บังคับเปิดใช้งาน** ก่อนเข้าหน้าผู้ดูแล · กู้คืนได้เมื่อทำอุปกรณ์หาย (ผู้ดูแลรีเซ็ตให้กัน + เครื่องมือ break-glass)
- **หน้า "ความปลอดภัย"** — ผู้ดูแล [/admin/security](app/(platform)/admin/security) และเจ้าของร้าน [/shop/security](app/(shop)/shop/security) เห็นไทม์ไลน์เหตุการณ์สำคัญ (เข้าสู่ระบบ, จัดการพนักงาน, กิจกรรมผิดปกติ) · เจ้าของร้านเห็นเฉพาะร้านตัวเอง
- **จัดการอุปกรณ์ที่เข้าสู่ระบบ** — ดูอุปกรณ์/IP ที่ยังล็อกอินค้าง แล้วออกจากระบบรายเครื่องหรือทุกเครื่องได้ (ทั้งผู้ดูแลและเจ้าของร้าน ที่หน้าตั้งค่า)
- **แจ้งเตือนความปลอดภัย** — ล็อกบัญชีอัตโนมัติเมื่อถูกเดารหัสผ่านรัวๆ พร้อมแจ้งเจ้าของบัญชี + ผู้ดูแล · ตรวจจับการเพิ่มแสตมป์ผิดปกติแล้วแจ้งร้าน + ผู้ดูแล
- **เจ้าของร้านดูแลพนักงานได้ดีขึ้น** — บังคับพนักงานออกจากระบบทุกอุปกรณ์ได้ (กรณีบัญชีหลุด)
- **ผู้ดูแล "ดูแบบร้าน" (อ่านอย่างเดียว)** — เข้าดูหน้าจอในนามร้านเพื่อช่วยเหลือ/แก้ปัญหา โดยแก้ไขข้อมูลไม่ได้ และมีบันทึกตรวจสอบ

### Changed
- **เพิ่มความเข้มของรหัสผ่าน** — ขั้นต่ำ 6 → 10 ตัวอักษร และตรวจสอบรหัสที่เคยรั่วไหล (HaveIBeenPwned) ทั้งตอนตั้งและเปลี่ยนรหัส
- เปลี่ยน/รีเซ็ตรหัสผ่าน = ออกจากระบบทุกอุปกรณ์โดยอัตโนมัติ (กันบัญชีที่ถูกขโมยใช้ต่อ)

### Security
- บันทึก audit เหตุการณ์สำคัญทั้งระบบ (เข้าสู่ระบบสำเร็จ/ล้มเหลว, รีเซ็ตรหัส, ระงับร้าน, ตรวจสลิป ฯลฯ)
- เพิ่ม HTTP security headers (HSTS, X-Frame-Options, CSP frame-ancestors, Referrer-Policy ฯลฯ) ทุกหน้า

### Fixed
- E2E: migrate ฐานข้อมูลทดสอบก่อนเซิร์ฟเวอร์บูต (แก้ CI e2e ที่เคยแดงจาก race)

## [1.12.0] - 2026-06-20

### Added
- **รีวิวร้านค้า** — ลูกค้าที่ผูกบัตรกับร้านแล้วให้ดาว (1–5) + เขียนคอมเมนต์ได้, เจ้าของร้านตอบกลับรีวิวได้ (แม้ร้านถูกพัก), แอดมินซ่อนรีวิวที่ไม่เหมาะสมได้ · คะแนนเฉลี่ยแสดงบนหน้าร้านสาธารณะและ popup บนแผนที่
- **รูปภาพร้าน** — เจ้าของร้านอัปโหลดรูปประจำร้าน, รูปปก (cover) และแกลเลอรีได้ · หน้า public ของร้านปรับเป็น ShopHero แบบโซเชียล (cover + รูปประจำร้านวงกลม)
- **หน้ารวมร้านค้าสาธารณะ [/shops](app/(public)/shops)** — ไดเรกทอรีร้านทั้งหมดสำหรับลูกค้าขาจร เข้าดูรีวิว/รายละเอียดร้านได้จากแผนที่หรือหน้ารวม
- **รายละเอียดร้านบนหน้า public** — คำอธิบายร้าน, เวลาเปิด-ปิด, ช่องทางติดต่อ และลิงก์โซเชียล (LINE/Facebook/Instagram/เว็บไซต์)

### Changed
- บังคับกฎ CSS ธีม semantic token ด้วย linter (ESLint กัน hardcode สีใน className + stylelint กัน hex ในชั้น token) — `npm run lint:all`

### Internal
- **ชุดทดสอบอัตโนมัติ** — test runner `node:test` + tsx (integration ขับ use case + Drizzle repo จริงบน in-memory libSQL ผ่าน DI container) + Playwright E2E smoke · `npm test` / `npm run test:e2e`
- **บังคับ Clean Architecture rules อัตโนมัติ** — dependency-cruiser คุมทิศทาง dependency 4 layer + `server-only` + ไม่มี cycle (`npm run depcruise` อยู่ใน `lint:all` + CI), ESLint tripwire เตือน component ที่แตะ DI container/repo, และ GitHub Actions CI (lint:all + tsc + test) block ทุก push/PR ที่ผิดกฎ — ลดการตรวจ manual
- refactor ให้ตรง Clean Architecture เป๊ะ (ย้าย business logic เข้า use case, projection helper เข้า application, เติม `server-only` ครบ)

## [1.11.0] - 2026-06-19

### Added
- **เพิ่มลีดจากแผนที่ (map-first) + ดึงข้อมูลร้านอัตโนมัติ** — หน้าใหม่ [/admin/leads/new](app/(platform)/admin/leads/new): ค้นหาย่าน เลื่อนแผนที่ แล้วแตะหมุดร้าน ระบบจะดึง ชื่อ/หมวดหมู่/เบอร์/ที่อยู่/พิกัด จาก OpenStreetMap มากรอกให้ เหลือแค่ตรวจและบันทึก · ถ้าร้านไม่มีในแผนที่ แตะจุดบนแผนที่เพื่อดึงที่อยู่ (reverse geocode) แล้วพิมพ์ชื่อเอง · ใช้ OSM ฟรี (Overpass + Nominatim) ไม่ต้องมี API key
- **`npm run gen:cron-secret`** — สคริปต์สุ่มค่า `CRON_SECRET` สำหรับตั้งใน Vercel (cron เตือนนัดติดตามลีด)

### Changed
- หน้า [/admin/leads](app/(platform)/admin/leads) เปลี่ยนฟอร์มเพิ่มลีดแบบเดิมเป็นปุ่ม “+ เพิ่มลีด” ที่ลิงก์ไปหน้าเพิ่มลีดจากแผนที่

## [1.10.0] - 2026-06-19

### Added
- **ระบบลีด (CRM ภาคสนาม) สำหรับแอดมิน** — เก็บข้อมูลร้านเป้าหมายที่จะออกไปสำรวจหาลูกค้า แยกจากร้านจริงในระบบ (ไม่ยุ่งกับการคิดเงิน/สถิติ) ที่ [/admin/leads](app/(platform)/admin/leads)
  - **แผนที่ลีด** — หมุดลงสีตามสถานะ (ใหม่/เข้าพบแล้ว/สนใจ/ปิดการขายได้/ไม่สำเร็จ) กดนำทางด้วย Google Maps ได้จากหมุด
  - **เก็บข้อมูลภาคสนาม** — ปักหมุดด้วยปุ่มดึงตำแหน่ง GPS, อัปโหลดรูปร้าน, บันทึกเบอร์/ที่อยู่/หมวดหมู่
  - **บันทึกการเข้าพบ** — ไทม์ไลน์ปฏิกิริยาแต่ละครั้งที่ไปคุย พร้อมเลื่อนสถานะการขายอัตโนมัติ
  - **แปลงเป็นร้านจริง 1 คลิก** — เมื่อปิดการขายได้ สร้างร้าน + บัญชีเจ้าของ + รอบทดลองใช้ และยกพิกัด/ที่อยู่ไปเป็นสาขาแรกให้ ร้านจึงขึ้นบนแผนที่สาธารณะทันที
  - **เตือนนัดติดตาม** — ตั้งวันนัด แล้วระบบเตือนแอดมินอัตโนมัติทุกวัน (ในแอป + LINE) เมื่อถึงกำหนด

## [1.4.0] - 2026-06-16

### Added
- **เข้าสู่ระบบด้วย OTP เท่านั้นสำหรับบัญชีที่ผูก LINE** — เจ้าของร้าน/พนักงานที่ผูก LINE แล้วต้องเข้าระบบผ่านรหัส OTP ทาง LINE (ปิดการใช้รหัสผ่านเพื่อความปลอดภัย); ผู้ดูแลระบบยังใช้รหัสผ่านได้เป็นทางสำรอง
- **แจ้งเตือนทาง LINE เมื่อเข้าสู่ระบบสำเร็จ** ทุกครั้ง (บอกเวลา/วิธี — หากไม่ใช่เจ้าของบัญชีจะรู้ตัวทันที)
- **ปุ่ม "ติดต่อผู้ดูแล" บนหน้าเข้าสู่ระบบ** สำหรับผู้ที่เข้าระบบไม่ได้ — ส่งเรื่องถึงผู้ดูแลทันที (แจ้งในแอป + LINE) เพื่อให้รีบช่วยเหลือ
- ป้องกันสแปม/บอทบนฟอร์มติดต่อสาธารณะ: CAPTCHA (Cloudflare Turnstile, เปิดเมื่อตั้งค่า key) + honeypot + จำกัดจำนวนครั้งต่อ IP

### Changed
- เพิ่มการจำกัดอัตราการเรียก (rate-limit) ต่อ IP สำหรับการขอ OTP และการล็อกอินด้วยรหัสผ่าน (กัน OTP-bombing / เดารหัสผ่าน); หน้าเข้าสู่ระบบตั้ง noindex

## [1.3.0] - 2026-06-16

### Added
- **เข้าสู่ระบบจากบัญชีที่เคยใช้บนเครื่องนี้** (แบบ Facebook) — หน้า login จำบัญชีที่เคยเข้าสำเร็จ แตะเลือกได้เลยไม่ต้องพิมพ์อีเมล (ยังต้องยืนยันตัวตนด้วย OTP/รหัสผ่านตามเดิม; เก็บแค่อีเมล+บทบาทในคุกกี้ httpOnly ไม่เก็บรหัสผ่าน/เซสชัน) พร้อมปุ่มลบบัญชีออกจากเครื่อง
- ปุ่มหน้าแรกรู้สถานะการเข้าสู่ระบบ: ถ้าเข้าระบบอยู่แล้วลิงก์ตรงไปแดชบอร์ด (ไม่วนผ่านหน้า login) + แสดงสถานะกำลังโหลดทันทีที่กด + เพิ่ม loading ให้หน้า login
- **พนักงานสาขา (staff) ใช้งานครบเทียบเท่าร้าน/แอดมิน** — เพิ่มแถบเมนูล่าง, หน้าตั้งค่า, หน้าแจ้งเตือน (+กระดิ่ง) และเชื่อมต่อ LINE ได้ (เดิมพนักงานเชื่อม LINE ไม่ได้)
- โหมดนักพัฒนา (เฉพาะเครื่อง local): สลับเข้าสู่ระบบเป็นผู้ใช้ใดก็ได้แบบคลิกเดียวเพื่อทดสอบ (กันหลุด production สองชั้น)

### Changed
- ปรับส่วนเชื่อมต่อ LINE เป็น **"ช่องทางเชื่อมต่อ & ความปลอดภัย"** สื่อชัดว่าใช้ทั้งรับแจ้งเตือนและเข้าสู่ระบบด้วย OTP (วางโครงรองรับช่องทางอื่นในอนาคต) ครบทุกบทบาท
- เปลี่ยนลูกศรข้อความ ←/→ เป็นไอคอน (lucide) ให้คมและสม่ำเสมอทุกหน้า
- ธีม retro: หัวข้อใช้ฟอนต์ Noto Serif Thai (อารมณ์นิตยสาร รองรับภาษาไทย) — เดิมอ้างฟอนต์ที่ไม่มีตัวไทยและไม่ถูกโหลด

## [1.2.1] - 2026-06-15

### Fixed
- แก้ขอบการ์ดสว่างผิดปกติใน dark mode — เปลี่ยนเส้นขอบการ์ดจาก `ring-brand-100` (สีพีชอ่อนคงที่) มาใช้สีขอบตามธีม (`ring-border`) ให้มืดกลมกลืนใน dark mode ทั้ง 3 ธีม

## [1.2.0] - 2026-06-15

### Added
- เข้าสู่ระบบด้วย OTP ผ่าน LINE (passwordless) สำหรับบัญชีที่ผูก LINE แล้ว — กรอกอีเมล → รับรหัส 6 หลักทาง LINE → ยืนยัน (OTP เก็บแบบ hash, อายุ 5 นาที, ล็อกหลังใส่ผิด 5 ครั้ง, ขอใหม่ได้ทุก 60 วิ) พร้อมทางสำรอง "ใช้รหัสผ่านแทน"
- หน้าประวัติการเติมวัน (ledger ของวันใช้งานที่ได้รับ) ในหน้าบิล
- หน้า **สถิติร้าน** (/shop/analytics): การ์ดตัวเลข (แสตมป์/แลกรางวัล/ลูกค้าแอคทีฟ/อัตราการแลก/ลูกค้าใหม่) + กราฟแนวโน้มรายวัน (recharts) + แยกตามประเภทแสตมป์/สาขา + ลูกค้าสะสมสูงสุด เลือกช่วง 7/30/90 วัน
- หน้า **สถิติระบบ** (/admin/analytics) สำหรับผู้ดูแล: ภาพรวมข้ามทุกร้าน (แสตมป์/แลกรางวัล/ลูกค้าแอคทีฟ/อัตราการแลก/ลูกค้าใหม่/ลูกค้าทั้งหมด/ร้านทั้งหมด/ร้านที่มีกิจกรรม) + กราฟแนวโน้มรายวันรวม + leaderboard ร้านค้าที่แอคทีฟสูงสุด เลือกช่วง 7/30/90 วัน

### Changed
- รายการที่ยาว (ประวัติแลกรางวัล, การแจ้งเตือน, คิวชำระเงิน, รายชื่อลูกค้า, คำขอติดต่อ) เปลี่ยนเป็นโหลดแบบ cursor "โหลดเพิ่ม" แทนการตัดจบที่จำนวนคงที่
- เก็บงานเล็กๆ: คอมโพเนนต์ `<Textarea>` ร่วม, รวมฟอร์แมตวันที่/ป้ายข้อความให้สม่ำเสมอ, เพิ่ม loading skeleton ที่ยังขาด

## [1.1.0] - 2026-06-15

### Added
- รองรับ **แสตมป์หลายประเภทต่อร้าน** (เช่นแบ่งตามราคา 10/20/30฿) — แต่ละประเภทมีเกณฑ์ครบและของรางวัลของตัวเอง สะสมแยกกัน พนักงานเลือกประเภทตอนกดแสตมป์ (ร้านประเภทเดียวพฤติกรรมเหมือนเดิม)

## [1.0.0] - 2026-06-15

รุ่น production แรก — ระบบบัตรสะสมแสตมป์ดิจิทัลสำหรับร้านค้าหลายสาขา (ปิดเฟส 2)

### Added
- บัตรสะสมแสตมป์ผ่านการสแกน QR สำหรับลูกค้า (ไม่ต้องพกบัตร) และระบบหลายสาขา
- ระบบบิลแบบ "เติมวัน" (day top-up) + แจ้งสลิป PromptPay + แอดมินตรวจอนุมัติ
- แผนที่ร้านค้าหน้าแรก + ปุ่มระบุตำแหน่งปัจจุบัน (ลูกค้า) และปักหมุดสาขา (ร้านค้า)
- ประวัติการแลกของรางวัล (ฝั่งร้านค้าและลูกค้า)
- ระบบแจ้งเตือนในแอป (กระดิ่ง + หน้า inbox) และ LINE OA push สำหรับ admin/เจ้าของร้าน
- ช่องทางร้านค้าติดต่อแอดมิน พร้อมสถานะคำขอและการแจ้งเตือนกลับเมื่อแอดมินรับเรื่อง
- แสดงเลขเวอร์ชั่น + commit ที่ท้ายทุกหน้า

### Changed
- เปลี่ยนเมนูฝั่งร้านค้า/แอดมินเป็น bottom tab bar + เมนู "เพิ่มเติม" ทุกขนาดจอ
- ราคาแพ็กเกจเติมวันสำเร็จรูปคิดจาก "ราคา/วัน" ของแต่ละร้าน (แทนราคาตายตัว)
- เพิ่ม loading skeleton ทุกหน้าให้รู้สึกลื่นขึ้นระหว่างโหลด

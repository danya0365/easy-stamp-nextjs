Phase 6 (2026-06-15, on `develop`, unreleased): shop owner analytics dashboard at **/shop/analytics**.

- **UI:** stat tiles (stamps issued, redemptions, active customers, redemption rate, new customers, total customers) + **recharts** daily trend ([DailyTrendChart.tsx](src/presentation/components/analytics/DailyTrendChart.tsx), `ComposedChart` bars+line, client) + CSS-bar breakdowns by stamp type / branch ([BreakdownList.tsx](src/presentation/components/analytics/BreakdownList.tsx), server) + top customers. Range selector 7/30/90 days = plain `<Link>` to `?range=N` (server re-render, no client). Shared [StatCard.tsx](src/presentation/components/ui/StatCard.tsx) (dashboard refactored to use it). Nav "สถิติ" (BarChart3) added to `NAVS.shop` overflow in [AppTabBar.tsx](src/presentation/components/layout/AppTabBar.tsx).
- **Data:** new read-model **`IAnalyticsRepository`** + [DrizzleAnalyticsRepository](src/infrastructure/repositories/drizzle/DrizzleAnalyticsRepository.ts) (kept analytics OUT of entity repos). Aggregates on `stamp_transactions` (earn `sum(quantity)`) + `reward_redemptions` (count) + `customers`. **Bangkok-day bucketing = `strftime('%Y-%m-%d', created_at, '+7 hours')`** (verified works on Turso with `...Z` ISO). Uses existing `(shopId, createdAt)` indexes — **no migration**.
- **Logic:** pure [analytics.ts](src/domain/services/analytics.ts) (`bangkokDay`, `rangeToSince`, `buildDailySeries` gap-fill, `redemptionRate`, `RANGE_OPTIONS=[7,30,90]`) — tsx-unit-tested (6 tests). Orchestrated by `GetShopAnalyticsUseCase`. Added Analytics* view types to [entities](src/domain/entities/index.ts). Wired `container.analyticsRepository`.
- **Dep added:** `recharts` 3.8.1 (supports React 19; charts are `"use client"`).
- **Scope/guard:** shop_owner only; page is under the (shop) billing guard (suspended/past-due shops redirect to /shop/billing). Not done: export. Relates to [[cursor-pagination]], [[multi-stamp-types]].

**Platform-admin cross-shop analytics (2026-06-15, on `develop`, unreleased):** added **/admin/analytics** (platform_admin), built on the SAME Phase-6 foundation. Reuses the pure domain service `analytics.ts` (`buildDailySeries`/`rangeToSince`/`redemptionRate`/`RANGE_OPTIONS`) and the `DailyTrendChart`/`BreakdownList`/`StatCard` components AS-IS. New pieces: separate read-model **`IPlatformAnalyticsRepository`** + `DrizzlePlatformAnalyticsRepository` (mirrors the per-shop Drizzle repo but DROPS the `shopId` filter — queries span all shops, bounded only by `gte(createdAt, since)`); `GetPlatformAnalyticsUseCase` (use-cases/platform/); types `PlatformAnalyticsSummary` (adds totalShops + activeShops=`count(distinct shopId)` earn-in-range), `AnalyticsShopRow`, `PlatformAnalyticsView`; wired `container.platformAnalyticsRepository`. UI: 8 StatCards + daily trend (all shops) + `byShop` leaderboard (top 15, name=shop, fed into `BreakdownList`). Nav "สถิติ" (BarChart3) added to `NAVS.admin` (now 6 items → overflows into "เพิ่มเติม"). **Migration 0008** (additive): `index(createdAt)` on stamp_transactions + reward_redemptions so cross-shop range scans don't fall back to the (shopId,createdAt) indexes. Verified: tsc/eslint 0, build ok, SQL reconciles (platform earn=366 = 93+91+91+91; activeShops=4/5), dev e2e at range 7/90 renders, /login guard ok.
